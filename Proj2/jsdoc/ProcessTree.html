processInformation()

This function is recursive. The arguments are the current node/leaf, the current material, texture and matrix (from heritage). The current node is the child node from the parent. If it's a leaf, then the material, texture and matrix are pushed to the leaf (to a special field created with this intent). The matrix is computed through the nodes. So, when it reaches a leaf, it has all the transformations required. If it's a node, then the matrix is computed and the textures and materials are calculated (according to the tree rules). This way of saving the matrix, textures and materials is far more efficient when the tree is complex. This computing is only done once.
